# AI模型项目文档导航指南

## 1. 文档指南概述

### 1.1 目的
本指南旨在为AI系统提供全面的项目文档导航框架，明确文档审查的最佳顺序，标识包含特定信息的文件，并提供精确的文件路径，使AI系统能够独立导航项目的信息架构，高效定位所需资源。

### 1.2 目标受众
- AI开发模型
- AI系统集成商
- 自动化文档分析工具
- 需要快速了解项目的新团队成员

### 1.3 文档结构
本指南按照从高层面到细节的逻辑顺序组织，帮助AI系统逐步深入理解项目：
1. 项目概述与愿景
2. 需求与规格说明
3. 架构与设计
4. 开发流程与指南
5. 测试与质量保证
6. 专业技术文档
7. 资源定位器（快速参考）
8. AI系统导航技巧

## 4. AI系统导航技巧

### 4.1 高效文档检索
- **关键词搜索**：使用精确关键词定位信息，如"API规范"、"数据模型"、"敏捷流程"
- **层级导航**：按照文档审查顺序逐步深入，从高层面到细节
- **交叉引用**：利用文档间的关联关系，从一个文档链接到相关文档

### 4.2 信息提取策略
- **结构化信息**：表格、列表等结构化内容优先提取
- **关键段落**：重点关注文档的标题、副标题和第一段
- **视觉元素**：图表、架构图等可视化内容包含重要信息

### 4.3 理解上下文关系
- **前后文关联**：结合多个文档理解完整上下文
- **版本一致性**：注意文档更新日期，优先使用最新文档
- **术语统一**：参考文档一致性修复计划，确保术语理解正确

### 4.4 资源更新与维护
- **定期检查**：定期检查文档目录，了解是否有新文档添加
- **版本控制**：注意文档的更新历史和版本号
- **变更跟踪**：关注文档中的更新日志，了解变更内容
